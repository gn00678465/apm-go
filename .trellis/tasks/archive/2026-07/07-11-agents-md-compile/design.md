# Design: apm-go `compile` — 最小 agents-family 子集

> **核准 — 2026-07-11**。research/findings.md「需拍板事項」6 點均依建議值確認：
> 1. 落地範圍=B（最小 agents-family 子集）；2. v1 target 集=(b) antigravity+
> codex+opencode；3. compile 與 install 關係=(b) 獨立指令；4. hand-authored
> 根 AGENTS.md 覆蓋策略=(a) 對齊 oracle 無條件覆蓋（(b) marker 保護留作候選
> 加固，見 §6）；5. A/B 基準線=oracle 固定
> `--single-agents --no-links --no-constitution`，normalize Build ID/APM
> Version/CR 三行；6. v1 flags=(a) 僅 `-t/--target`。實作依此定案完成，見
> `.trellis/spec/backend/compile-contract.md`。

## 1. Goals / Non-Goals

### Goals

- 新增 `apm-go compile` 指令:把 local `.apm/instructions/*.instructions.md` 與
  `apm_modules/<key>/.apm/instructions/*.instructions.md` 編譯成**專案根單一 `AGENTS.md`**。
- 語意對照 oracle:`uv run apm compile --single-agents --no-links --no-constitution -t <agents-family>`。
- 支援 target:`antigravity`、`codex`、`opencode`(apm-go 現有的三個 agents-family adapters;
  Python 對照 `compile_family="agents"`,targets.py:613/685/708)。
- Idempotent:同輸入重跑 byte-identical、內容不變時不重寫(oracle 實測 sha256 相同)。
- 不改任何既有 install/deploy 行為。

### Non-Goals(documented deviations,A/B 腳本以斷言記錄)

- distributed 佈局(context_optimizer)、子目錄 AGENTS.md、orphan cleanup(--clean)
- claude / gemini / vscode compile families(CLAUDE.md、GEMINI.md、copilot-instructions.md)
- constitution 注入、markdown link 解析、chatmode、`--watch`/`--validate`/`--root`/`--dry-run`
- vscode dedup(`can_dedup_agents_md_instructions` 只對 vscode 生效,target_detection.py:309-335,
  agents family 永不 dedup → 本子集天然無此邏輯)
- managed_section 模式、user-scope compile(PRD non-goal)

## 2. CLI 契約

```
apm-go compile [-t|--target <name[,name…]>]
```

- **target 解析**:重用 `deploy.ResolveTargets`(flag > apm.yml target: > 目錄訊號自動偵測,
  adapter.go:76-116)。解析結果過濾出 agents-family 集合 {antigravity, codex, opencode}:
  - 至少一個 agents-family target → 編譯一次,輸出一份根 AGENTS.md(多個 agents-family
    同選不重複輸出——Python 同構:同 family 只走一次 `_compile_agents_md`)。
  - 解析結果**只含**非 agents-family(claude/copilot)→ 硬錯誤 exit 2:
    `compile for target(s) X not implemented in apm-go yet`。理由:Python 對 claude 會產出
    CLAUDE.md,apm-go 靜默成功會是 zero-output silent success(Python #820 反模式)。
  - 解析結果為空(無訊號)→ 同上訊息 + exit 2(Python 此情境 fallback 到 vscode/minimal,
    apm-go 不支援該 family → 顯式拒絕)。
- **專案 gate(對齊 cli.py:347-385)**:
  - 無 `apm.yml` → stderr `Not an APM project - no apm.yml found` + exit 1。
  - 無可編譯內容(無 `apm_modules/`、`.apm/` 無 `*.instructions.md`)→ error + exit 1。
- exit codes:0 成功(含 idempotent no-op)、1 專案 gate/寫檔失敗、2 不支援的 compile family。

## 3. 資料流

```
manifest.Load(apm.yml)
  → deploy.ResolveTargets(flag, manifest.Targets, projectDir)   // 既有
  → filter agents-family;非法情境依 §2 exit
  → collect instructions(重用 deploy 收集順序,deploy.go:72-118):
      local .apm/instructions/*.instructions.md(先)
      直接依賴宣告序 apm_modules/<key>/.apm/instructions/(後)
      transitive(lockfile 排序)最後
      同名衝突:先到先贏(local 贏、先宣告贏;對齊 discovery.py:181-184)
  → parse 每檔 frontmatter:
      applyTo 值(scalar 原樣字串;YAML list 取第一個元素,對齊 parser.py:95-119)
      body = frontmatter 之後全文(claudeFrontmatterRE 同款 regex,instructions_claude.go:13)
      body strip 後為空 → 過濾(template_builder.py:74,81)
  → render(§4)→ Build ID 固化(§5)→ idempotent write(§6)
```

**注意**:分組 key 用 **raw applyTo 字串**(不拆逗號)——不可重用 install 端 `parseApplyTo`。
`applyTo: "**/src/**, **/api/**"` 產生單一 `## Files matching \`**/src/**, **/api/**\`` 組。

## 4. 輸出格式契約(oracle 實測 2026-07-11,apm_cli 0.21.0)

```
# AGENTS.md
<!-- Generated by APM CLI from .apm/ primitives -->
<!-- Build ID: <12-hex> -->
<!-- APM Version: <version> -->

## Global Instructions

<!-- Source: <relpath> -->
<body strip 後>
<!-- End source: <relpath> -->

## Files matching `<pattern>`

<!-- Source: <relpath> -->
<body strip 後>
<!-- End source: <relpath> -->

---
*This file was generated by APM CLI. Do not edit manually.*
*To regenerate: `apm compile`*
```

- 排序:Global 組在前(組內按 relpath 字典序);pattern 組按 pattern 字典序,組內按 relpath
  (template_builder.py:59-84)。每條 instruction 之後跟一個空行。
- relpath = 相對 projectDir 的 forward-slash 路徑(local:`.apm/instructions/x.instructions.md`;
  依賴:`apm_modules/<key>/.apm/instructions/x.instructions.md` — **實作前先 probe oracle 確認**,
  見 implement.md step 0)。
- 檔尾:最後一行後有一個換行(sections 以 "" 結尾 join)。
- **行結尾:apm-go 輸出 LF**(repo 慣例,同 instructions_claude.go);Python 在 Windows 寫 CRLF
  → documented deviation,A/B normalize `\r`。
- **APM Version 行**:寫 apm-go 自己的版本 → documented deviation,A/B normalize 該行。

## 5. Build ID(對齊 build_id.py:22-39)

1. 內容先放 placeholder 行 `<!-- Build ID: __BUILD_ID__ -->`。
2. `lines = content.splitlines()`;去掉 placeholder 那一行;其餘以 `\n` join,SHA256,取 hex 前 12 字。
3. placeholder 行替換為 `<!-- Build ID: <hash> -->`;保留原 trailing newline。
4. 雜湊在 LF 域計算(寫檔前),與行結尾 deviation 無關。

## 6. 寫檔與 idempotency

- 產生最終內容後,若根 `AGENTS.md` 已存在且 byte-equal(LF 域比較)→ 不寫,
  印 `No changes detected; preserving existing AGENTS.md for idempotency`(對齊 cli.py:721-723 可觀察行為)。
- 否則:寫 temp 檔 + rename(對齊 output_writer.py atomic 語意)。
- **覆蓋策略(拍板 #4,建議 (a))**:對齊 oracle full 模式——無條件整檔覆蓋,
  不做 hand-authored marker 檢查(Python 單檔模式亦無;managed_section 才有保護,不在 v1)。
  候選加固 (b):無 `<!-- Generated by APM CLI` marker 時拒寫 + 警告——記錄於此,待上游行為
  變更再跟進。
- **測試安全**:apm-go repo 根有手寫 AGENTS.md;所有測試/A/B 一律在 t.TempDir()/TEMP scratch 執行。

## 7. 邊界與相容性

- 新增物:`cmd/apm/compile.go` + `internal/compile/`(template、buildid、frontmatter 解析可
  從 `internal/deploy` 抽公用或最小複製——傾向把 `claudeFrontmatterRE` 提升為共用,避免雙份)。
- 不觸碰:`internal/deploy` 部署行為、`internal/manifest` 偵測、install/uninstall/update 流程。
- spec 落點:新開 `.trellis/spec/backend/compile-contract.md`(輸出契約 + deviations 清單),
  antigravity-target-contract.md 增補一行互見。

## 8. Rollback 形狀

純增量:revert = 刪 `cmd/apm/compile.go`、`internal/compile/`、A/B 腳本、spec 檔;
無 schema/lockfile/設定遷移;不影響既有指令。唯一使用者可見副作用是 compile 寫出的
AGENTS.md 檔案本身(使用者可手刪)。

## 9. 驗收(對照 PRD AC)

- A/B:`evals/ab_agents_compile.py` — 兩邊 scratch 專案(local + dependency instructions、
  global/scoped/comma/brace/空 body 各案),oracle 跑
  `compile --single-agents --no-links --no-constitution -t antigravity`,Go 跑 `compile -t antigravity`;
  比對 AGENTS.md(normalize:Build ID 行、APM Version 行、`\r`)+ 各自 Build ID 重算自洽 +
  idempotency(重跑 byte-identical)+ exit codes(無 apm.yml → 兩邊 1;claude-only → Go 2 為
  documented deviation)。
- unit ≥ 80%(internal/compile);`go build ./... && go vet ./... && go test ./...` 綠。
- deviation 記錄:A/B 腳本 docstring + spec 檔雙落點。
