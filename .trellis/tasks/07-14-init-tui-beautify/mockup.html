APM-Go 終端機介面美化計畫

為 apm-go 導入一套統一的終端機視覺主題，重整目前散落各檔、手刻前綴的 互動選單與 stdout 輸出。以單一 charmbracelet 生態承接：lipgloss（樣式 / 表格 / 邊框 / 顏色） + huh（互動表單 / spinner，建於 bubbletea），共用同一組色票與符號。v1 的 pterm 方案已作廢—— pterm 的寬度引擎無法正確渲染對齊的 box-drawing 表格，且顏色是全域旗標，有 per-writer 洩漏 footgun。

研究來源：Context7 · charmbracelet/lipgloss v2 · charmbracelet/huh v2 現況：0 依賴・0 共用 helper 分支：feat/init-tui-lipgloss

01統一主題 — APM Terminal Theme

單一色票，huh 與 lipgloss 共讀同一組色碼：訊息 / 表格 / box 皆由 lipgloss.NewStyle().Foreground(...) 套用，huh Theme 套用相同色碼於互動元件。色碼皆對得上 ANSI，非 TrueColor 終端會自動降階（lipgloss colorprofile 原生處理，見下方降階策略）。

brand#2dd4bf
heading#8aa0ff
success#3fb950
warning#d29922
error#f85149
muted#8b949e

符號系統（取代目前 [+] [i] [!] [warn] [>] [*] 混用）

✓ success・原 [+] [*]
ℹ info・原 [i]
! warning・原 [!] [warn]
✗ error・原 stderr 裸字串
▸ progress・原 [>] → spinner
• list item・原 - / |--
降階策略（lipgloss 原生 per-writer）：lipgloss.Fprint(w, s) / Fprintln(w, s) 會把 w 包進 colorprofile.NewWriter(w, os.Environ())， 依該 writer 自己的終端能力自動降階：TrueColor → ANSI256 → ANSI16 → mono → strip， 非 TTY / NO_COLOR / CI 下自動去色；huh 互動元件另有 accessible 模式。統一封裝進新的 internal/ux package，各檔案既有的私有 print* helper 改呼叫它。
對比 pterm（棄用原因之一）：pterm 的顏色是全域旗標 （pterm.DisableColor()），所有 writer 共用同一份設定——先對 TTY 印過、再對檔案 writer 印時容易漏關色（per-writer footgun）。lipgloss 每次呼叫都依該 writer 自己判定 profile， 無全域狀態、無洩漏風險。
串流契約（opus advisor 修正）：不是把所有輸出 blanket 搬到 stderr —— 許多指令的「結果行」本來就在 stdout（marketplace list/validate、pack、install/uninstall 摘要）。 規則改為逐處保留原串流：把該行原本的 os.Stdout/os.Stderr 原樣傳給 ux.*。 互動元件固定走 stderr+stdin。驗收保護所有指令的 stdout 結果內容不變，不只 normalize。

02套件策略 — 已定案：charm 生態（lipgloss + huh）

v1 的 pterm 方案已作廢：pterm v0.12.83 的寬度引擎無法正確處理「對齊的 box-drawing 表格」 （多位元組分隔線算錯寬度導致跑版），且顏色是全域旗標，有 per-writer 洩漏 footgun。v2 全部收斂到 charm.land/lipgloss/v2（樣式 / 表格 / 邊框 / 顏色）+ charm.land/huh/v2 （互動表單 / spinner，已建於 bubbletea），單一生態、移除 pterm 相依。

能力v1（已棄用）v2（採用）
訊息 Success/Info/Warn/Errorpterm PrefixPrinterlipgloss Style + Fprint(w, …)
表格 Tablepterm.DefaultTablelipgloss lipgloss/table（RoundedBorder）
條列 BulletListpterm.BulletListPrinterlipgloss lipgloss/list
巢狀 Treepterm.TreePrinterlipgloss lipgloss/tree
Box（確認框）pterm.Boxlipgloss RoundedBorder
進度 Spinnerpterm spinnerhuh huh/spinner
互動 Confirm/Input/Password/MultiSelecthuh（不變）huh（不變）
關鍵優勢：lipgloss Fprint(w, …) 依該 writer 自己的 colorprofile 原生 downsample / 去色——徹底取代先前手刻的 renderForWriter，且沒有全域旗標 footgun。 huh 已建於 bubbletea，改用 lipgloss 使全部相依收斂到 charm 生態，go mod tidy 後 go.mod 無 pterm。

05落地方式

  • 非互動 / --yes 路徑維持原樣純文字輸出（不套動畫 spinner）。
  • 測試：internal/ux 加單元測試（關色時輸出純文字 golden）；互動流程既有 e2e 以 --yes 繞過，維持綠燈。
